TShock Quest System

Introduction

Welcome to the TShock Quest System plugin. Originally created by Twitchy and now maintained by Ijwu. For a tutorial on how the system works go here.

It's assumed you know the basic concepts programming and of the object oriented paradigm. You may want to look into some other online tutorials.

Config Files

Every quest script must have a config file with it. Config files must have the .cfg extension and must contain at least the Name field to show up in the Quest System. See config file into here.

Changelog

     
  • Version 2.1 - April, 28, 2013
  • Added CallFunction trigger.
    • Arguments: LuaFunction func, LuaTable args
    • Calls the function passed to it when evaluated with the table passed to it as the argument.
  • Lowered default tick rate to 1 ms.
  • Added Enqueue and ClearQueue functions.
    • Enqueue takes a list of triggers and adds them to the quest in that order.
    • ClearQueue clears the list of triggers in the quest.
  • Changed the name of some triggers to be more specific to their function:
    • Private to MessagePlayer
    • Gather to GatherItem
    • Pass to Delay
      • Delay is now useless as the pause is 1ms long. I'll look into other ways to pause a quest.
    • DeleteBoth to DeleteTileWall
    • RetrieveItem to GiveUpItem
    • Hunt to HuntMob
  • Triggers are now dynamically loaded from the Triggers namespace.
    • Other plugins should now be able to register their own triggers with the system.
      • Custom triggers must be a subclass of QuestSystemLUA.Trigger.
        Custom triggers must have only one constructor.

Known Bugs

     
  • GiveUpItem spams the player with "change" when they drop too many items if they keep moving around.
    • As a result this trigger is now disabled.
  • If a quest script never adds any triggers, it crashes on start.
